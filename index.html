You Have A Nice Day

Always code as if the guy who ends up maintaining your code will be a violent psychopath who knows where you live

They think learning a programming language is the same as learning to program. That is a profound mistake when it comes to learning to program. At first, don’t bother trying to learn every feature of your language, learn variables, if statements, and while loops, you can write practically anything by understanding those 3 things. Don’t learn a language, learn to program! Learn how to learn. 90% of the time, that means knowing how to use Google. Before you ask anything spend 5 minutes trying to find out on your own. Break it down. Programming is basically addressing one big problem (write Minecraft, for example) as thousands of little problems. Don’t think “How will I write Minecraft”, think “How will I get a window on the screen” and then address the next problem, just take it one step at time.

Don’t “practice”. Do! Programming isn’t something you practice, it’s something you do. Don’t sit down with a book and try to remember everything in it, you should be writing software. Pick a language, get the IDE/editor/compiler or whatever, and start. Write something small. Get the gist of ifs/whiles/variable etc. and write a simple game like Minesweeper.

Trying to learn more than language programming simultaneously Though it is possible, it is not easy. I tried to learn C++ and Python simultaneously. I did learn the basic syntax of both the languages, but i did not learn the language specific important fundamentals like the memory management, data structures and core concepts specific to a programming language. Not sticking to one programming language in the initial phase of learning. Its human nature to want everything he finds interesting. Its a good sign, but in the initial phase of your learning this can lead you off course. I started learning Python, when i was halfway through i read somewhere about JavaScript and found that to be interesting and so i put my Python learning on hold and switched to JS. This went on for some time. I was working as a manual tester that time and there was scope to move into automation testing team. Now i had to learn Java so that i can learn Selenium(automation testing tool). I had learned three different languages but none completely. This was a big mistake, DO NOT COMMIT this mistake.
